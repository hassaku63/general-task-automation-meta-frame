# task-review: タスクレビュースキル

指定タスクの4文書を読み込み、品質チェックリストで評価する。

## 使い方
```
/task-review {タスク名}
```

## 実行手順

### 1. 文書の読み込み
1. `$ARGUMENTS` からタスク名を取得する。引数がない場合はユーザーに聞く
2. `tasks/{task-name}/` 配下の4文書（PLAN.md, SPEC.md, TODO.md, KNOWLEDGE.md）をすべて読み込む
3. 文書が存在しない場合はその旨を報告し、`/new-task` での作成を案内する

### 2. チェックリストで評価

以下の観点で各文書を ○（十分）/ △（改善余地あり）/ ×（不十分）で評価する。

#### PLAN.md
- [ ] ゴールが具体的で測定可能か
- [ ] 現状の課題が明確か
- [ ] 理想の状態がゴールと整合しているか
- [ ] 制約・前提が漏れていないか

#### SPEC.md（最重視）
- [ ] インプット/アウトプットが明確か
- [ ] **自動化の境界が明確に定義されているか**
- [ ] AI 担当部分と人間担当部分が適切に分離されているか
- [ ] 判断ポイントの基準が具体的か
- [ ] 処理フローに抜け漏れがないか
- [ ] 受け入れ基準が検証可能か

#### TODO.md
- [ ] タスクが十分に分解されているか
- [ ] フェーズ分けが適切か
- [ ] 完了条件が明確か

#### KNOWLEDGE.md
- [ ] 実装後の場合: 学びが記録されているか
- [ ] 再利用可能な知見が特定されているか

### 3. レビュー結果の報告

以下の形式で報告する:

```
## レビュー結果: {タスク名}

| 文書 | 評価 | コメント |
|------|------|----------|
| PLAN.md | ○/△/× | ... |
| SPEC.md | ○/△/× | ... |
| TODO.md | ○/△/× | ... |
| KNOWLEDGE.md | ○/△/× | ... |

### 改善提案
1. ...
2. ...

### 総合判断
実装に進めてよいか / 修正が必要か
```

### 4. 改善提案
- 評価が △ または × の項目について、具体的な改善案を提示する
- 特に SPEC.md の「自動化の境界」が曖昧な場合は、明確化を強く推奨する
