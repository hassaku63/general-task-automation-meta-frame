# new-task: タスクスキャフォールドスキル

新しい自動化タスクのワークスペースを対話的に作成する。
同梱のテンプレートをもとに4文書を生成し、ヒアリングを通じて内容を埋めていく。

## テンプレート

- [PLAN テンプレート](templates/PLAN.md) - ゴール・課題の言語化
- [SPEC テンプレート](templates/SPEC.md) - 仕様の精緻化（自動化の境界を含む）
- [TODO テンプレート](templates/TODO.md) - タスク管理
- [KNOWLEDGE テンプレート](templates/KNOWLEDGE.md) - 知見の蓄積

## 使い方
```
/new-task {タスク名や説明}
```

## 実行手順

### 前準備
1. `$ARGUMENTS` からタスク名を取得する。引数がない場合はユーザーに聞く
2. タスク名をケバブケースに正規化する（例: `勤怠報告の自動化` → `attendance-report`）
3. `tasks/{task-name}/` ディレクトリを作成する
4. 上記テンプレート4文書を読み込み、`tasks/{task-name}/` に配置する。`{タスク名}` プレースホルダーをタスク名に置換する

### Phase 1: PLAN.md 作成
5. ユーザーに以下をヒアリングする（音声入力でざっくり話してもらうことを推奨）:
   - 何を達成したいか（ゴール）
   - 今何に困っているか（現状の課題）
   - どうなっていれば嬉しいか（理想の状態）
   - 制約や前提はあるか
6. 回答を PLAN.md に整理して書き込む
7. ブレインダンプセクションに、ユーザーの生の発言や思いつきをそのまま記録する

### Phase 2: SPEC.md 精緻化
8. PLAN.md をもとに SPEC.md のドラフトを作成する
9. 特に「自動化の境界」を重点的にユーザーと議論する:
   - 各ステップについて AI/人間 どちらが担当すべきか
   - 人間の判断が必要なポイントはどこか
   - その判断基準は何か
10. インプット/アウトプット、処理フロー、受け入れ基準を確定させる

### Phase 3: TODO.md 生成
11. SPEC.md の処理フローをもとにタスクを分解する
12. フェーズ（調査→実装→テスト）ごとに整理する
13. 優先順位をユーザーに確認する

### 完了
14. 作成した4文書の概要をユーザーに報告する
15. 次のステップとして `/task-implement {task-name}` を案内する
